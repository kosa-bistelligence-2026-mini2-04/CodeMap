🎯 작업 목적
현재 랜딩 페이지의 사용성을 대폭 개선하고, 분석 화면과 채팅 화면을 '통합 워크스페이스' 형태로 개편하여 사용자 경험(UX)을 향상시키기 위함입니다. 
빈 랜딩 페이지의 이탈률을 줄이고, 사용자가 저장소를 분석하고 에이전트와 대화하는 과정을 직관적이고 끊김 없이 경험할 수 있도록 다수의 시각적 편의 기능(파일 트리, 자동완성, 다이어그램 등)을 도입했습니다. 또한 부자연스러웠던 번역체 UI 문구를 자연스럽게 다듬었습니다.

✨ 주요 변경 사항
**[UI/UX 개선 및 신규 기능 도입]**
- [x] **레포 검색 자동완성 (PROJECT-REPO-F-201)**: URL 직접 입력 대신 키워드 입력 시 GitHub Search API와 연동하여 자동 추천되도록 입력 UX 대폭 향상
- [x] **Trending Repos 섹션 추가**: 랜딩 페이지에 인기 레포 카드를 배치하여 원클릭("이 레포 분석해보기")으로 분석을 유도
- [x] **로컬 폴더 탐색 UI**: 랜딩 페이지 `cm-input`에서 로컬 경로 입력 시, Mac/Windows 호환 가능한 OS 기본 폴더 탐색기를 통해 폴더를 지정할 수 있도록 UI 개선
- [x] **랜딩 페이지 문구 개선 (의역)**: "감사" 등 어색한 직역 표현을 다듬어 자연스러운 워딩으로 전면 수정 ("CodeMap AI는 멀티 에이전트 인텔리전스로 전체 저장소를 수 초 만에 이해하는 초능력을 제공합니다" 등)
- [x] **파일 트리 사이드바 (RAG-PARSE-F-201)**: 분석 완료된 저장소의 디렉토리 구조를 사이드바에서 트리 형태로 탐색할 수 있도록 시각화
- [x] **Mermaid 다이어그램 생성 (RAG-GRAPH-F-201)**: 아키텍처 큰 틀, 의존성 그래프 등을 다이어그램으로 자동 렌더링
- [x] **대화 내보내기 (DOCS-GEN-F-201)**: 채팅 대화 및 분석 결과를 Markdown(.md) 파일로 다운로드하여 공유 및 보관할 수 있는 기능
- [x] **에이전트 탐색 과정 표시 (AGENT-SEARCH-B-204)**: 에이전트가 현재 읽고 있는 파일명 및 탐색 단계를 실시간으로 표시하여 투명성 및 신뢰도 향상

**[구조 및 로직 개선 (구현 순서 반영)]**
- [x] **UI 정리**: 중복 페이지 제목/앱 내부 실행 버튼 제거, 캐시 옵션은 고급 설정으로 이동, 모델 선택 자동화(단순화), Fast/Deep 명칭 명확화

🛠️ 리뷰 및 로컬 테스트 방법
1. 로컬 환경에서 `npm run dev` 실행 후 랜딩 페이지에 접속해 주세요.
2. 랜딩 페이지의 개선된 문구와 하단 **Trending Repos** 섹션이 잘 보이는지 확인합니다.
3. 입력창에 키워드(예: `react`)를 입력하여 **자동완성**이 잘 동작하는지, 로컬 아이콘을 클릭해 **OS 폴더 탐색기**가 잘 뜨는지 테스트해 주세요.
4. 임의의 저장소를 분석한 뒤, 다음 플로우를 검증해 주세요:
   - 우측 패널에서 에이전트 탐색 과정이 실시간으로 잘 뜨는지 확인
   - 에이전트 답변 내용의 출처 파일 클릭 시 해당 코드로 잘 이동하는지 확인
   - 우측 패널 상태에서 "전체 채팅 화면"으로 갔다가 "분석 화면"으로 돌아왔을 때 대화 내역이 잘 보존되는지 확인
   - 좌측 파일 트리 사이드바 및 Mermaid 다이어그램 렌더링 확인
   - 모바일 사이즈에서 중앙(코드뷰)/채팅 패널 간 탭 전환이 매끄러운지 확인
5. 새로고침 시에도 분석 및 대화 내용이 복구되는지 확인해 주세요.
- *참고: lint, typecheck, frontend build, backend API 테스트 모두 로컬에서 통과 확인했습니다.*

⚠️ 리뷰어에게 당부하는 점
- 랜딩 페이지의 카피라이팅이 기존 직역 텍스트보다 자연스러워졌는지 살펴봐주시고, 더 나은 표현이 있다면 적극적으로 코멘트 부탁드립니다!
- 통합 워크스페이스 형태로 바뀌면서 컴포넌트 간 **상태(State) 보존 로직**이 크게 변경되었습니다. 화면을 이리저리 이동할 때 대화 내역이나 파일 선택 상태가 유실되는 엣지 케이스가 있는지 중점적으로 리뷰해 주시면 감사하겠습니다.
- 로컬 폴더 탐색기의 경우 OS(Mac/Windows)별 환경 차이가 있을 수 있으니 한 번씩 클릭 테스트 부탁드립니다.
