# CodeMap: 지능형 소스코드 분석 및 대화형 AI 비서 서비스 기능 명세서

## 1. 서비스 개요 및 비전 (Service Overview & Vision)

### 1.1 배경 및 당면 과제
신규 프로젝트에 합류한 개발자나 대규모 소스코드를 다루는 엔지니어는 수많은 파일과 복잡하게 얽힌 코드 간의 유기적 관계를 파악하는 데 상당한 시간과 노력을 소모합니다. 소스코드의 구조적 파악이 늦어질수록 팀의 온보딩 비용이 증가하고 프로젝트 커뮤니케이션 효율이 떨어지며 개발 생산성이 저하됩니다.

### 1.2 해결책: CodeMap
CodeMap은 GitHub 저장소(Repository)의 코드를 깊이 이해하고 분석하여 사용자의 질문에 답을 제공하는 **지능형 소스코드 대화형 비서**입니다. 사용자가 GitHub 저장소의 URL 주소를 입력하는 것만으로 복잡한 초기 설정 없이 즉시 분석이 시작되며, 소스코드의 문맥적 의미를 파악하여 사람과 대화하듯 명확한 설명과 아키텍처 정보를 제공합니다.

### 1.3 핵심 가치 및 지향점 (Core Value)
- **온보딩 시간 단축**: 수십만 줄의 낯선 코드베이스를 단 몇 분 만에 시각화 및 문서화하여 파악할 수 있도록 돕습니다.
- **의사결정 효율 향상**: 변수명이나 개발 기술명을 정확히 모르는 기획자나 초임 개발자도 일상어(자연어)로 질문하고 소스코드 상의 정확한 기능 위치를 안내받을 수 있습니다.
- **답변 신뢰성 보장**: 단순한 텍스트 매칭이나 근거 없는 추측(환각)을 방지하고, 실제 코드 구조와 참조 관계를 역추적하여 사실에 입각한 답변과 출처(파일명, 줄 번호)를 명확하게 제공합니다.

## 2. 4대 핵심 기능 상세 명세 (Core Features)

### 2.1 데이터 수집: 스마트 프로젝트 등록 (Git 클론 및 필터링)
* **기능 개요**: 사용자가 입력한 Git 저장소의 소스코드를 실시간으로 안전하게 복제하고 분석에 필요한 정보만 똑똑하게 걸러내어 저장하는 초기 단계입니다.
* **핵심 가치 및 UX 디자인**:
  - **원클릭 저장소 연동**: 사용자는 별도의 개발 환경 세팅 없이 브라우저 화면에서 분석하고자 하는 GitHub URL만 입력하고 클릭하면 자동으로 코드가 복제됩니다.
  - **지능형 노이즈 필터링**: 라이브러리 파일(`node_modules`, `.venv`), 빌드 산출물(`dist`, `build`), 컴파일러 캐시, 패키지 잠금 파일(`package-lock.json`, `poetry.lock`), 대용량 미디어 파일 및 바이너리(이미지, PDF) 등 코드 분석과 상관없는 노이즈를 자동으로 걸러냅니다. 단, 프로젝트 환경설정 파일(예: `.env.example`, `package.json`, `requirements.txt`) 및 안내 문서(예: `README.md`)는 신규 개발자를 위한 온보딩 대시보드 및 구동 가이드 생성에 필수적이므로 제외하지 않고 안전하게 보존합니다.
  - **분석 효율 극대화**: 노이즈를 걸러냄으로써 시스템 처리 속도를 비약적으로 단축시키고, AI 비서가 오직 순수한 비즈니스 로직과 설계 문서에만 100% 집중하여 답변 품질을 높일 수 있게 합니다.

### 2.2 지식 기반: 코드 맥락 및 관계망 이해 (RAG 및 코드 임베딩)
* **기능 개요**: 단순 단어 비교 검색(Keyword Search)의 한계를 뛰어넘어, 코드가 갖는 고유한 역할과 설계 의미, 그리고 파일 간의 연결 관계를 학습하여 똑똑한 '의미 기반 사전'을 구축하는 단계입니다.
* **핵심 가치 및 UX 디자인**:
  - **타겟 우선 분석 (Targeted Priority Analysis)**: 사용자가 특정 폴더나 파일(예: "src/auth 폴더에 대해 설명해줘")을 지정하여 질문할 경우, 전체 저장소의 인덱싱이 완료될 때까지 기다릴 필요 없이 **해당 타겟만 0순위로 우선 분석**하여 답변을 즉시 제공합니다(TTFT 혁신적 단축). 나머지 코드베이스는 백그라운드에서 비동기적으로 처리됩니다.
  - **의미 단위 구조화**: 코드를 줄 단위의 단순한 텍스트가 아닌 클래스, 함수, 메서드 등 실제 소프트웨어 기능 단위로 조각내어 역할을 파악합니다.
  - **유기적 관계망 지도 구축**: 어떤 소스코드 파일이 다른 파일을 참조하고 있는지(의존성 및 가져오기(Import) 흐름)를 추적하여 유기적인 논리 지도를 구성합니다.
  - **계층적 개념 탐색**: 상위 폴더(디렉토리) 구조로 갈수록 하위 세부 요약본들을 자연스럽게 압축하여 체계적인 지식 구조를 구축합니다.
  - **자연어 질문 해석**: 사용자가 특정 변수명이나 함수명을 기억하지 못하더라도 평소 사용하는 언어로 질문하면, 시스템이 문맥을 이해하고 의도에 맞는 소스코드 파일을 찾아냅니다.

### 2.3 지능형 검색: 자율 탐색형 AI 코드 분석 (Agentic Search 및 도구 호출)
* **기능 개요**: AI 비서가 사용자의 질문을 단 한 번만 조회해 답변을 기계적으로 내놓는 것이 아니라, 숙련된 소프트웨어 엔지니어처럼 스스로 계획을 세우고 디렉토리를 열어보며 답을 찾아내는 자율형 탐색 기능입니다.
* **핵심 가치 및 UX 디자인**:
  - **자율 탐색 계획 수립**: 복잡한 다단계 질문을 받으면 스스로 질문의 의미를 분석한 후, *"디렉토리 구조 파악 -> 파일 검색 -> 소스코드 정독 -> 참조 파일 확인"* 순으로 논리적인 탐색 여정을 계획합니다.
  - **실시간 소스코드 추적**: 시스템에 구현된 코드 탐색 기능(도구)들을 주도적으로 실행하며 최적의 답변 단서를 찾습니다.
  - **자가 교정(Self-Correction) 프로세스 및 루프 한계 설정**: 첫 번째 탐색 과정에서 답을 찾지 못하거나 정보가 부족하다고 인지할 경우, 연계된 다른 파일 경로를 탐색하여 오류를 스스로 바로잡는 자가 검증을 진행합니다. 이때 무한 분석 루프에 빠져 답변이 지연되거나 불필요한 API 비용이 발생하는 것을 원천 방지하기 위해, AI 비서의 자율 탐색 도구 호출은 최대 5회, 처리 시간은 최대 20초로 제한합니다. 제한된 범위 내에서 최종 결론에 도달하지 못할 경우 수집된 정보를 최대로 종합하여 최선의 답변을 제공하고, 한계가 발생했음을 사용자에게 알립니다.
  - **명확한 출처 제공**: 최종 답변 시 분석의 근거가 된 소스코드 파일명과 줄 번호(Line Numbers)를 하이퍼링크 형태로 제공하여, 사용자가 직접 코드를 대조해 보며 결과를 즉각적이고 완벽하게 신뢰할 수 있게 합니다.

### 2.4 자동화된 문서화: 계층형 프로젝트 가이드북 자동 생성 (Map-Reduce 요약)
* **기능 개요**: 수십만 줄의 낯설고 방대한 저장소 코드를 일일이 읽지 않아도, 한눈에 파악할 수 있는 고품질의 Markdown 형식 '프로젝트 가이드북'을 자동으로 집필해 주는 기능입니다.
* **핵심 가치 및 UX 디자인**:
  - **상향식(Bottom-up) 정보 집필**: 개별 소스코드 파일의 요약을 먼저 작성하고(Map), 폴더별로 이 요약본들을 모아서 상위 설명을 작성한 뒤, 최종적으로 프로젝트 전체의 마스터 리포트로 융합(Reduce)하는 체계적인 요약 과정을 거칩니다.
  - **정보 누락 및 한계 돌파**: 많은 분량을 한 번에 분석할 때 나타나는 정보 유실이나 본문 생략 문제 없이, 모든 소스코드의 구조가 꼼꼼하고 완벽하게 설명서에 반영됩니다.
  - **아키텍처 전반의 시각화**: 저장소의 연동 작업이 완료됨과 동시에 폴더 구조 트리, 핵심 데이터 흐름, 환경별 구동 방법이 정리된 문서가 즉석에서 생성되므로 새로 팀에 합류한 멤버들의 온보딩 리소스를 획기적으로 줄여줍니다.

## 3. 사용자 경험 및 인터랙션 요구사항 (UX Requirements)

### 3.1 WebSocket 기반 실시간 분석 진행률 시각화 (Real-Time Progress Tracking)
* **사용 목적**: 프로젝트 등록 및 구조화 작업과 같이 대기 시간이 긴 비동기 프로세스 수행 중, 사용자가 느끼는 막연함과 이탈을 방지하고 시스템이 실시간으로 동작 중임을 투명하게 알립니다.
* **상세 기능 구성**:
  - **실시간 비동기 통신 피드백**: 양방향 실시간 통신 기술을 적용해, 서버에서 진행 중인 작업 상태를 지연 없이 화면에 지속 업데이트합니다.
  - **연결 끊김 방지 및 폴백(Fallback) 메커니즘**: 만약 실시간 통신(WebSocket) 연결이 끊어질 경우, 사용자 화면의 중단이나 불편 없이 표준 HTTP 요청을 통한 주기적인 확인 방식(예: 5초 주기 폴링)으로 자동 전환되어 실시간 분석 진행률을 계속 가져옵니다.
  - **단계별 체크리스트 방식**: 단순히 진행 바(%)만 보여주는 것을 넘어, 현재 수행 중인 내부적인 핵심 업무 흐름을 눈으로 확인할 수 있는 상세 체크리스트 뷰를 결합합니다.
    - *예시 UI 시각화*:
      - `[완료] 1단계: 저장소 소스코드 가져오기 (Git Clone 완료 - 2.5MB 수집)`
      - `[완료] 2단계: 불필요 노이즈 파일 제외 (38개 파일 제외 정제)`
      - `[진행중] 3단계: 코드 지식 구조화 및 맥락 분석 (진행률: 64% - 24/38개 파일 처리 중)`
      - `[대기] 4단계: 프로젝트 가이드북 자동 생성`

### 3.2 신규 프로젝트 온보딩 추천 대시보드 (Onboarding Recommendations Dashboard)
* **사용 목적**: 분석 완료 후 첫 화면에 진입한 사용자가 챗봇을 어떻게 활용하고 어떤 파일부터 읽어야 할지 몰라 헤매지 않도록 구체적인 방향을 제시합니다.
* **상세 기능 구성**:
  - **기술 스택 및 언어 분석 결과**: 감지된 소스코드의 프로그래밍 언어 비율과 사용된 대표 프레임워크 정보를 시각적인 차트로 제공합니다.
  - **우선 권장 소스코드 Top 10**: 프로젝트 전체의 의존성 구조와 논리적 중요도를 판단하여, 신입 개발자가 가장 먼저 정독해야 할 핵심 엔트리포인트 파일(라우터, 주요 비즈니스 컨트롤러 등) 10개를 선정하고 해당 파일의 경로와 한 줄 핵심 역할 설명을 목록으로 추천합니다.
  - **자동 구동 및 빌드 퀵 가이드**: 소스코드 내 구성 설정 파일을 스스로 분석하여 로컬 환경에서 시스템을 실행하는 방법(예: `npm run dev`, `python main.py` 등)을 똑똑하게 추론하여 대시보드 내 퀵스타트 섹션으로 제공합니다.

### 3.3 대화형 인터페이스 및 실시간 응답 스트리밍 (Streaming Chat with Citations)
* **상세 기능 구성**:
  - **실시간 타이핑 스트리밍**: AI 비서가 생각한 답변을 화면에 표시할 때 전체 긴 텍스트를 대기했다가 한 번에 출력하지 않고, 생성되는 즉시 글자 단위로 자연스럽게 흘려보내 체감 반응 속도를 크게 높입니다.
  - **에이전트 행동 메시지 중계**: 대화 중 AI 비서가 추가 탐색 도구를 호출하는 동안 대화창 상단에 *"로그인 관련 모듈 auth.py 파일을 분석하는 중..."*과 같은 구체적인 행동 메시지를 간결하게 노출하여 진행 상황을 투명하게 알립니다.

## 4. 이용 범위 및 제약사항 (Boundaries & Limits)

### 4.1 시스템 지원 한계 및 제외 조건
원활한 고속 분석 속도 유지와 리소스 비용 관리를 위해 다음과 같은 이용 범위를 규정합니다.
* **최대 지원 소스코드 파일 수**: 단일 Git 저장소 기준 최대 100개 파일로 제한합니다. (프로젝트 성격에 따라 관리자 설정으로 확장 가능) 만약 불러온 저장소의 파일 수가 100개를 초과하는 경우, 단순 실패 처리하는 것이 아니라 다음과 같은 사용자 편의 기능을 제공합니다.
  - **사용자 선택형 디렉토리 트리 제공**: 사용자가 직접 웹 화면의 폴더/파일 트리 구조를 통해 분석에 포함하거나 제외할 폴더 및 파일을 수동으로 선택 및 해제할 수 있도록 지원합니다.
  - **중요 파일 자동 선별 및 경고**: 혹은 파일 간의 참조 및 가져오기(Import) 흐름을 자동으로 분석하여 가장 핵심이 되는 100개의 파일만 지능적으로 자동 선택해 분석을 진행하고, 사용자에게 일부 파일이 분석에서 제외되었음을 경고 메시지로 안내합니다.
* **단일 파일 최대 용량 제한**: 개별 소스코드 파일당 최대 100KB까지만 가져옵니다. (미니파이 가공된 무거운 파일이나 대량의 테스트 데이터 덤프 등으로 인해 발생하는 불필요한 리소스 낭비와 무한 분석 루프 현상을 원천 방지)
* **자동 분석 배제 대상 (Exclusion List)**:
  - 외부 모듈 및 패키지 보관소 (`node_modules`, `.venv`, `packages`, `vendor` 등)
  - 빌드 산출물 및 임시 실행 저장소 (`dist`, `build`, `bin`, `out`, `__pycache__` 등)
  - 패키지 버전 잠금 파일 및 단순 기록용 로그 (`package-lock.json`, `yarn.lock`, `poetry.lock`, `*.log` 등)
  - 각종 비텍스트 바이너리 파일 (이미지, PDF, 동영상, 폰트, 압축 파일 등)

### 4.2 보안 및 프라이버시 보호 정책
* 다운로드 및 복제된 소스코드는 활성 사용자 세션과 연계된 암호화된 임시 저장 공간(Encrypted Ephemeral Storage)에 안전하게 보관되어 실시간 코드 검색 및 대화형 분석 기능을 지원합니다. 이 소스코드는 사용자가 세션을 종료하거나 30분 동안 아무런 활동이 없는 비활성화 상태가 되면 복구 불가능한 방식으로 완전히 영구 삭제됩니다.
* 최종적으로 저장 및 장기 보호되는 데이터는 코드의 유기적인 관계를 요약한 지식 관계망 정보뿐이며, 사용자의 핵심 소스코드가 제3자에게 노출되거나 외부 서비스로 유출되지 않도록 전 경로 암호화를 적용합니다.
